Life FAQ

  • Q. What do the buttons do?

    Q. What do I need to run Life?

    A. An HPC, with WindowsCE. This program was written on a Velo1, under
    WindowsCEv1. I've been told that it works on other HPC devices, but I haven't
    had the opportunity to test that myself. The program should run under WindowsCE v2,
    again I haven't been able to test this. Obviously I'm as keen as everyone else to get
    hold of WinCEv2. If you have a WinCEv2 machine, then installing the original
    MFCWCE.DLL file might mean the program will work.

    Q. There is a large blank portion of screen!

    A. There should be, if your HPC has more than 480 pixels across. My HPC
    has 480, and so I cannot test software which has a larger screen.

    Q. Do I have the latest version?

    A You do if you have version v1.3 (look in the Help box). This version adds new features
    to v1.1 If you have older versions, please delete them. I've discovered a bug in these which
    could potentially crash your HPC.

    Q. I get an error message when I try to run it!

    A. If you get an error message when you try to run the program, this is most likely due to
    you not having the MFC library file stored on your HPC. This is a file containing common
    functions, and is required by applications written in a particular way. You may already have
    this library depending on your HPC (the HP device has it in ROM) or if have already installed
    a program which includes the library. If you don’t have these files, you will have to download them.

    Q. What is Life anyway?

    A. Life is a cross between a computer game and a simulation. The screen displays consists of a
    grid where "cells" can exist. You can place cells anywhere on the grid by clicking with the pen.
    Once the cells are placed on the grid, you can see what happens in the next generation:
    click ">" or select "Go Once" from the menu. Cells live or die depending on two rules:

    1. If a cell has 2 or 3 neighbours, everything is fine and it survives. Any more than three and it
    starves to death, less than two and it dies of loneliness

    2. If an empty square has three cells around it, a new cell will be born.

    These rules are applied to every square in the grid for each generation. If you wish, you can
    click on ">>" or select "Start" from the menu, and successive generations will be calculated
    automatically. This will continue until all the cells are dead or you select "Stop". The speed
    of the generations is controlled in the "Speed" menu, and there are three settings.

    Q. So, this is interesting is it?

    A. Yes, because certain patterns produce very interesting results. You can set the speed
    to "fast", click "start" and watch as the screen fills with a swarm of cells.

    Q. Are you going to release an update?

    Q. Do you know of any other links to Life on the Web?

    A. Yes, loads. Any of these links will bring you to pages created by people who really,

    (which all have huge grids) and you find discussions on glider guns, spaceships and all
    sorts of other things, such as modelling real world problems using Cellular Automata.
    You'll also find plenty of little shapes to try in your copy of Pocket Life.
